On-Boarding HowTo: SDDC

Create an SDDC

SDDCs are created from the SDDC view of the vmc console. Click on “Create SDDC”.

  1. Specify the AWS Region, number of hosts, the host type, and SDDC name.
  2. If this is your first SDDC then you must establish a link to the customer-owned AWS account. You will be required to log into this AWS account. Please ensure that you have sufficient permissions to run the required CloudFormation template. You will not need to perform account linking again when deploying future SDDCs unless you need to link them to a different AWS account.
  3. Specify the VPC and subnet to use for SDDC cross-linking. Remember that the Availability Zone (AZ) of the subnet you choose will determine the AZ of the SDDC. For stretched-cluster deployments you will specify 2 subnets (and 2 AZs as a result).
  4. Provide the management subnet for the SDDC. Remember that the SDDC will utilize 100% of the address space which you provide for infrastructure. You may not use any part of this subnet for compute workloads.
  5. Acknowledge payment and deploy the SDDC.

Once the SDDC has been deployed then it will be visible from the SDDC view of the vmc console.

Assuming you have permitted inbound access through the SDDC gateway firewalls, you may access vCenter of the SDDC via its FQDN (which resolves to a public IP address by default).
